# AOP-NP-instances
鉴于现有标准算例库无法直接适配本文研究的多重图场景，本文基于经典 VRP 基准构建了包含 43 组测试算例的专用数据集，分为小规模与大规模两类，另设 5 组独立算例用于LrMA的个体效能评估。所有算例均遵循“X”命名规范，其中 X 代表节点规模。
data_small：该数据集基于Augerat等人（1995）所提出的VRP实例中包含站点数n=32-65个节点的小规模的算例。采取了以下设置来适用于本问题：(a)针对坐标求取欧几里得距离，然后生成距离矩阵，距离矩阵生成多重图中两条弧的分数与时间，时间优先弧（时间短、分数低）的分数（reij0）与时间（teij0）系数分别是0.1、0.1，分数优先弧（时间长、分数高）的分数（reij1）与时间（teij1）系数是0.3、0.2；(b)节点分数（pi）为随机生成，其中有30%的高分数站点，高分数站点的分数为100，低分数站点的分数为1，另外节点的游览时间统一设为0；(c)每个算例设置了总预算时间 T 为该算例的节点数。
data_big：该数据集基于Uchoa等人（2017）所提出的VRP实例，生成了包含101-701个节点的23个算例。除去 T 值设置为固定的1000外，其余数据的设计均同data_small。
一、算例生成
所有算例参考Vansteenwegen等[1]的数据选用方式，以Solomon经典数据集和Cordeau等的基准数据集[2]为基础生成ATOP的测试实例。其中，Small算例来源于基础算例rc102；Large算例中，40_1到40_3来自基础算例50r_101，60_1到100_3来自基础算例r100，120_1到120_3来自基础算例pr03和pr08，140_1到140_3来自基础算例pr04和pr09，160_1到160_3来自基础算例pr06和pr10。
算例的具体生成步骤如下：
（1）优先弧及连接弧比例设置：对于小规模算例，优先弧和连接弧的总比例在nn−1∗0.1, nn−1∗0.2内随机选取；对于大规模算例，优先弧和连接弧的总比例在(nn−1∗0.2, nn−1∗0.4]内随机选取，其中 n 代表骑行节点数量。在随机选取优先弧时，需满足以下两个条件：
a）任意节点至多作为一条优先弧的起点；
b）任意节点至多作为一条优先弧的终点；
c）所有优先弧线构成的有向子图中不包含任何环结构。
（2）旅行时间矩阵构建：该矩阵基于基础算例中的节点坐标计算得出。优先弧、连接弧以及高吸引力竞争弧的旅行时间数值设定为基础算例中两点之间的欧几里得距离；其余与每条高吸引力竞争弧平行的其他弧线，其旅行时间则设定为该高吸引力竞争弧旅行时间的二分之一。
（3）吸引力分数矩阵构建：高吸引力竞争弧的吸引力分数设为基础算例中连接节点对的需求值之和；其余与每条高吸引力竞争弧平行的其他弧线，其旅行时间则设定为该高吸引力竞争弧旅行时间的二分之一；优先弧和连接弧的吸引力分数均设为0。

二、算例格式
第1行：
骑行节点数量，旅行时间阈值

第2行：
用作标记旅行时间矩阵（无实际意义）；

第3至第n+3行：（n为骑行节点数）
优先弧、连接弧和高吸引力竞争弧的旅行时间矩阵。括号中的数值表示该高吸引力竞争弧所对应平行弧的旅行时间。

第n+4行：
用作标记吸引力分数矩阵（无实际意义）；

第n+5至2n+5行：
优先弧、连接弧和高吸引力竞争弧的吸引力分数矩阵。其中，字母“r”和“a” 用于标识优先弧和连接弧，分数均为0。其余数值表示高吸引力竞争弧的吸引力分数，括号中的数值为该高吸引力竞争弧对应平行弧的吸引力分数。
